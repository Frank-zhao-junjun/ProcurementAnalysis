# 预警处置规则

版本: v1.0
更新时间: 2026-04-11
适用模块: 首页驾驶舱、支出分析、需求趋势、供应源分析

## 定义

P0 预警用于标记需要在晨会或日内跟进的高风险事项，要求记录责任链路和预计恢复时间。

## 纳入项

- 预警编号
- 风险等级
- 责任人 Owner
- 当前状态 Status
- 预计完成时间 ETA
- 处置建议

## 排除项

- 无责任归属的纯观察类提示
- 无法确认数据来源的临时异常
- 已关闭且不再跟踪的历史事项

## 示例问答

Q: P0 预警是否必须带责任人？
A: 必须。每条 P0 预警至少要带一名业务责任人 Owner。

Q: 当前状态要记录哪些值？
A: 至少记录待确认、处理中、已缓解、已关闭四类 Status，并附 ETA。

## 常见误解

- 误解: 只要有风险就可以列为 P0。
  澄清: P0 预警必须同时满足风险影响显著且需要明确处置动作。
- 误解: 有责任人就不需要 ETA。
  澄清: ETA 是晨会追踪的基本字段，没有 ETA 不能视为完整闭环。
