**版本**: v2.1  
**日期**: 2026-04-11  
**状态**: 已通过五方评审与安全评审，待Coze数据训练确认后进入开发  
**替代文档**: `2026-04-11-ai-assistant-integration-design.md`, `2026-04-11-ai-assistant-integration-design-v2.md`

## 1. 概述

### 1.1 背景

采购经营分析驾驶舱 Phase 1 已完成 8 大模块、37 个 ECharts 图表、10 个 KPI 和 6 条 P0 预警的可视化建设。现阶段缺口不在"有没有数据"，而在"用户是否能快速得到可执行结论"。

v1 设计已验证 AI 集成方向可行，但在以下方面存在高风险：

- 页面上下文仅在发送消息时采集，状态容易漂移
- 将整页 KPI 和预警全量注入 system prompt，存在 token、准确率和数据外发风险
- 代理层缺少明确的超时降级、高可用和缓存策略
- 业务口径文档不足以支撑追问型场景
- 日志治理和第三方数据边界定义不完整

v2 的目标是把以上问题在设计阶段先收口，再进入开发。

v2.1 在 v2 基础上补充了安全合规细节（日志留存、访问控制、隐私提示、Coze数据边界），满足"有条件通过"要求。

### 1.2 目标

通过 Coze Bot + 后端代理 + 最小上下文注入 + 按需取数机制，实现以下能力：

1. **智能问数**：支持围绕 8 大模块和 37 个图表的自然语言问答。
2. **智能解读**：基于当前视图和业务规则生成结论、原因和建议。
3. **上下文感知**：AI 感知当前模块、筛选器、选中对象和图表焦点。
4. **口径可追溯**：涉及降本、TCO、预警、供应商等问题时，可回溯到业务口径文档。
5. **安全可控**：敏感数据最小化传输，日志可脱敏审计，第三方数据边界明确，用户隐私提示完整。

### 1.3 范围

**本阶段覆盖**

- 驾驶舱页面内嵌 AI 助手组件
- 支持 8 大模块、37 个图表的智能问答与解读
- 支持 10 个 KPI、6 条 P0 预警的上下文问答
- 支持快捷提问、回答引用来源、页面联动提示
- 支持最小上下文注入、按需取数、缓存、超时降级
- 支持日志治理、分级访问、隐私提示、数据边界控制

**本阶段不覆盖**

- 自动发起采购流程或审批流
- 跨系统写操作，例如自动补货、自动下单
- 开放式多代理协同分析
- 移动端独立 App

## 2. 设计原则

### 2.1 最小必要原则

不向模型全量发送整页数据，只发送回答当前问题所需的最小上下文。

### 2.2 状态优先原则

AI 的页面理解必须来源于实时同步状态，而不是发送消息瞬间的单次抓取。

### 2.3 口径先于回答

涉及金额、比例、价格偏差、TCO、预警判断等问题时，先遵循业务口径，再组织回答。

### 2.4 降级优先原则

当 Coze、网络或代理异常时，优先给用户稳定反馈，而不是长时间空等或返回不透明错误。

### 2.5 安全默认收紧

默认不记录原始问题全文，不向第三方传输非必要业务数据，不暴露内部错误细节，日志分级留存与访问控制。

### 3.2 核心设计决策

#### 决策 1：采用"最小 system prompt + 按需工具取数"

不再把 `current_kpis`、`p0_alerts` 整页 JSON 直接塞进 system prompt。system prompt 只保留角色、回答规范、边界约束。实时业务数据由代理层工具接口按需返回。

#### 决策 2：在前端增加状态同步总线

新增 `dashboardStateBus`，统一接收 Tab 切换、筛选器变化、图表点击和 KPI 高亮事件。AI 助手使用总线中的最新状态，而不是每次现抓 DOM。

#### 决策 3：代理层承担降级控制

AI 代理不仅转发请求，还负责缓存、限流、熔断、trace_id、脱敏日志和错误标准化。

#### 决策 4：业务口径作为独立知识层

把降本、物流、返利、TCO、价格偏差、预警责任等定义从图表文档中抽离成独立规则文档，避免回答时口径漂移。

#### 决策 5：日志分级与隐私保护

日志分级留存（访问180天/错误90天/分类365天），默认脱敏，分级访问控制，审计记录完整。

## 4. 前端设计

### 4.1 组件形态

AI 助手继续采用右下角悬浮组件，但增加 3 种状态：

1. `collapsed`: 仅保留入口按钮
2. `dock`: 380 x 500 对话窗口
3. `focus`: 临时扩展高度，适合长回答阅读

组件默认采用深色科技风，颜色变量沿用驾驶舱设计系统，但命名统一为 `--pa-ai-*` 前缀，避免样式污染。

### 4.2 页面状态同步

#### 4.2.1 状态对象 schema

```javascript
{
  session: {
    userRole: 'procurement_manager',
    dateRange: '2026-W15'
  },
  ui: {
    activeTab: 'tab2',
    activeModule: '支出分析',
    selectedDimension: 'category',
    activeFilters: {
      bu: '事业部3',
      subsidiary: null,
      category: '钢材'
    },
    selectedEntity: {
      type: 'chart',
      id: 'chartPriceDeviation'
    },
    highlightedChart: 'chartPriceDeviation'
  }
}
```

#### 4.2.2 状态更新事件

以下事件必须触发状态同步：

- Tab 切换
- 维度切换
- KPI 卡片点击
- 图表点击或 hover 高亮
- 筛选器变化
- AI 触发的页面跳转或高亮操作

#### 4.2.3 前端接口设计

```javascript
dashboardStateBus.update(partialState)
dashboardStateBus.getSnapshot()
dashboardStateBus.subscribe(listener)
```

AI 组件发送消息前只读取 `getSnapshot()`，不直接遍历 DOM 现抓状态。

### 4.3 前端生命周期与性能

#### 4.3.1 生命周期要求

- 页面初始化时创建 AI 助手单例
- 页面卸载时销毁 SDK 实例和订阅监听器
- Tab 切换时不重建 AI 组件，只更新状态
- AI 高亮图表前先校验图表实例未销毁

#### 4.3.2 内存与性能约束

- 不得在每次提问时重新初始化 SDK
- 不得在每次回答后重复绑定事件
- 不得在不可见图表上频繁执行高亮动画
- AI 组件对页面帧率影响需控制在可接受范围内

#### 4.3.3 大屏适配

- 1920 固定宽度下默认停靠右下角
- 小屏或浏览器缩放低于阈值时自动切换为 `collapsed`
- 避免覆盖右侧关键图表区域；若冲突，优先收起对话框

### 4.4 快捷提问重构

快捷按钮从"演示型按钮"改为"晨会型按钮"，建议保留 4 类：

1. `本周最差 3 项指标`
2. `当前 P0 预警怎么处理`
3. `这张图的结论和原因`
4. `给我一条晨会汇报口径`

### 4.5 隐私提示设计

## 5. 对话与 Prompt 设计

### 5.1 Prompt 分层

#### 5.1.1 System prompt

只保留长期稳定规则：

- 角色：采购经营分析驾驶舱 AI 助手
- 输出语言：中文
- 格式规范：金额、比例、来源标注
- 边界约束：不确定时明确说明，禁止编造数据
- 回答顺序：结论、原因、建议、来源

#### 5.1.2 Runtime context

只包含当前会话必要状态：

- 当前模块
- 当前 Tab
- 当前筛选器摘要
- 当前选中对象摘要

#### 5.1.3 Tool data

当用户问题涉及具体 KPI、预警或图表明细时，由代理调用工具接口返回结构化数据。

### 5.2 推荐回答模板

```markdown
结论：本月降本额为 156.8 万元，当前达成率 108.1%。

原因：钢材谈判降幅扩大，同时事业部 3 集中采购带来价格优势。

建议：优先跟进塑料粒子缺货风险，并补充核对该口径是否包含物流费用。

来源：降本分析模块 / 月度降本追踪图 / 降本口径文档 v1
```

### 5.3 幻觉控制规则

当出现以下情况时，必须禁止模型直接给结论：

- 实时数据缺失
- 当前口径文档不存在
- 用户问题跨模块但未给足筛选条件
- 实时数据与知识库示例冲突

此时回答策略为：

1. 明确说明缺失项
2. 告知需要切换模块或补充条件
3. 只引用已确认的数据和来源

## 6. 工具与数据访问设计

### 6.1 工具接口定义

代理层提供以下工具能力给 Coze：

1. `get_dashboard_kpis`
2. `get_dashboard_alerts`
3. `get_chart_snapshot`
4. `get_business_glossary`
5. `get_faq_answer`

### 6.2 示例：图表快照工具

```json
{
  "tool": "get_chart_snapshot",
  "input": {
    "chart_id": "chartMonthlySavings",
    "active_tab": "tab1",
    "filters": {
      "bu": "事业部1"
    }
  }
}
```

### 6.3 工具返回约束

- 返回结构化 JSON，不返回冗长自然语言
- 金额、比例、单位字段必须拆开
- 必须带 `source_id` 和 `version`
- 缺失数据时显式返回 `not_available`

## 7. 后端代理设计

### 7.1 服务职责

代理层职责定义如下：

1. JWT 鉴权和来源控制
2. Prompt 组装与工具路由
3. 缓存、限流、熔断、超时降级
4. 脱敏日志、trace_id、监控埋点
5. 错误码标准化
6. 日志分级留存与分级访问控制

### 7.2 高可用建议

- 多实例部署
- 负载均衡入口
- 健康检查接口
- 缓存使用 Redis
- 限流与熔断状态共享

### 7.3 超时与降级策略

| 场景 | 阈值 | 处理策略 | 前端提示 |
|------|------|----------|---------|
| Coze 响应慢 | > 5 秒 | 展示思考中状态 | "正在分析，请稍候" |
| Coze 超时 | > 10 秒 | 进入降级模式 | "AI 分析超时，先返回简版结论" |
| 连续失败 | 5 次 / 5 分钟 | 熔断 30 秒 | "AI 服务暂不可用，请稍后重试" |
| 高频标准问题 | 命中缓存 | 返回缓存答案 | "已返回最近一次分析结果" |

### 7.4 缓存策略

适合缓存的问题：

- KPI 定义类问题
- 图表解释类问题
- FAQ 标准问题
- 更新频率较低的晨会总结问题

不适合缓存的问题：

- 当前筛选器下的实时追问
- 刚触发的预警处理状态
- 带用户个性化上下文的问题

缓存键建议：

`sha256(question + active_tab + filters + glossary_version)`

### 7.5 可观测性

每次请求必须打通统一的 `trace_id`，并记录：

- 请求时间
- 响应时间
- 命中模块
- 工具调用次数
- 缓存是否命中
- 是否触发降级
- 用户反馈结果

## 8. 安全与合规设计

### 8.1 数据最小化

- 禁止发送完整页面状态到第三方平台
- 禁止默认发送全部 KPI 与全部预警详情
- 仅在需要时发送与问题直接相关的数据片段

### 8.2 密钥与身份认证

- Coze API Key 存于服务端密钥管理系统或环境变量
- 建议后续接入企业级 KMS / Vault
- JWT 必须包含过期时间，并限制前端存储策略
- 启用 CORS 白名单、HTTPS、HSTS

### 8.3 日志治理

#### 8.3.1 日志分级与留存周期

| 日志类型 | 留存周期 | 理由 | 到期后处理 |
|---------|---------|------|-----------|
| **访问日志** (trace_id, 用户ID, 模块, 时间) | 180天 | 满足安全审计和问题追溯需求 | 自动归档至冷存储，保留2年后删除 |
| **错误日志** (错误类型, 堆栈, 上下文摘要) | 90天 | 足够定位问题，减少敏感数据暴露时长 | 自动删除 |
| **对话分类日志** (问题类型, 命中率, 反馈结果) | 365天 | 支持AI效果分析和知识库优化 | 匿名化后保留，删除用户关联 |
| **原始请求日志** | **禁止记录** | 防止敏感业务数据泄露 | N/A |

#### 8.3.2 日志到期后处理策略

**自动删除** (访问日志/错误日志) :
```python
# 示例：定时清理任务 (Node.js/Python)
schedule('0 2 * * *', async () => {
  const cutoff = new Date();
  cutoff.setDate(cutoff.getDate() - RETENTION_DAYS);
  
  await LogCollection.deleteMany({
    type: { $in: ['access', 'error'] },
    created_at: { $lt: cutoff }
  });
});
```

**匿名化归档** (对话分类日志) :
- 删除用户ID、IP地址、会话标识
- 保留问题类型、准确率、响应时间等统计字段
- 匿名化后的数据用于模型优化和报表生成

**冷存储** (访问日志超180天) :
- 压缩后迁移至低成本存储 (AWS Glacier/阿里云OSS归档)
- 保留2年后彻底删除
- 需要法务审批才能提前访问

#### 8.3.3 分级访问权限

| 角色 | 可访问日志 | 访问方式 | 审批要求 |
|------|-----------|---------|---------|
| 运维工程师 | 访问日志、错误日志 (脱敏) | 只读控制台 | 直属经理审批 |
| 安全审计员 | 全部日志 (脱敏后) | 审计专用界面 | 安全负责人审批 |
| 开发人员 | 错误日志 (脱敏后) | 调试工具 | 开发组长审批 |
| 业务人员 | **无** | N/A | N/A |

#### 8.3.4 审计记录

所有日志访问必须记录审计日志：
```json
{
  "audit_event": "log_access",
  "access_time": "2026-04-11T10:30:00Z",
  "accessor_id": "user_123",
  "accessor_role": "security_auditor",
  "access_type": "query",
  "query_range": "2026-04-01 to 2026-04-10",
  "approval_id": "approval_456",
  "ip_address": "10.0.1.100"
}
```

#### 8.3.5 日志默认不记录内容
