## 1. 日志留存与生命周期管理

### 1.1 日志分级与留存周期

| 日志类型 | 留存周期 | 理由 | 到期后处理 |
|---------|---------|------|-----------|
| **访问日志** (trace_id, 用户ID, 模块, 时间) | 30天 | 控制敏感数据暴露窗口，满足近期问题追溯需求 | 自动删除 |
| **错误日志** (错误类型, 堆栈, 上下文摘要) | 30天 | 满足问题定位需要，避免长期保留潜在敏感摘要 | 自动删除 |
| **对话分类日志** (问题类型, 命中率, 反馈结果) | 30天 | 保持与统一日志治理周期一致 | 自动删除 |
| **原始请求日志** | **禁止记录** | 防止敏感业务数据泄露 | N/A |

### 1.2 日志到期后处理策略

**自动删除** (访问日志/错误日志/对话分类日志) :
```python
# 示例：定时清理任务 (Node.js/Python)
schedule('0 2 * * *', async () => {
  const cutoff = new Date();
  cutoff.setDate(cutoff.getDate() - RETENTION_DAYS);
  
  await LogCollection.deleteMany({
    type: { $in: ['access', 'error', 'conversation_classification'] },
    created_at: { $lt: cutoff }
  });
});
```

**统一策略** :
- 所有允许记录的日志保留 30 天
- 到期后自动删除，不做冷存储归档
- 不保留可回溯个人或业务实体的长期日志副本

## 3. Coze 平台数据使用边界确认

### 3.1 待确认事项清单

**必须向 Coze 商务/技术支持确认的问题**：

1. **模型训练** ：通过 API 发送的采购数据、KPI、预警信息是否会被用于改进 Coze 基础模型？
2. **数据存储** ：API 请求和响应内容是否在 Coze 服务端持久化存储？存储多久？
3. **跨境传输** ：数据是否会传输到境外服务器处理？涉及哪些区域？
4. **数据隔离** ：企业级账号是否提供数据隔离保障？其他租户能否访问我们的数据？
5. **合规认证** ：Coze 平台是否通过等保、ISO 27001、SOC 2 等安全认证？

### 3.2 建议的确认方式

| 方式 | 优先级 | 负责方 | 时间 |
|------|--------|--------|------|
| 查阅 Coze 企业版服务协议 | P0 | IT经理 | 1天内 |
| 邮件咨询 Coze 技术支持 | P0 | IT经理 | 2天内 |
| 要求 Coze 提供数据处理说明函 | P1 | 法务+IT | 1周内 |
| 查阅 Coze 安全白皮书 | P1 | 安全专家 | 3天内 |

### 3.3 临时缓解措施（确认前）

在获得 Coze 确定性结论前，实施以下缓解措施：

```markdown
1. **数据最小化**：继续严格执行 v2 方案的最小上下文原则，只发送必要数据
2. **敏感字段脱敏**：所有供应商名称、具体金额在发往 Coze 前脱敏
   - "利安德巴塞尔" → "供应商A"
   - "156.8万元" → "约150万元量级"
3. **测试数据先行**：先用测试数据跑通全流程，验证功能后再接入生产数据
4. **合同条款**：在 Coze 服务合同中增加数据使用限制条款
```

### 3.4 当前已知结论与未决项

基于当前项目输入，先采用以下约束：

- **API 输入数据是否用于模型训练**：按当前结论，记为“不会”。
- **API 请求/响应是否会被平台持久化存储，以及存多久**：当前项目阶段暂不考虑，视为未完成外部核验项。

因此，本附录可关闭“模型训练”相关表述风险，但**不能关闭平台存储周期与持久化边界风险**。

- **当前状态**：未取得平台侧正式结论。
- **接受理由**：
  - 当前方案已执行最小必要传输，不发送整页状态。
  - 当前方案已限制日志记录范围，且本地日志仅保留 30 天并自动删除。
  - 当前方案已确认“API 输入数据不会用于模型训练”。
  - 当前阶段可先以脱敏、测试数据优先和最小上下文方式启动开发工作。

### 6.2 放行边界

本次放行仅适用于以下工作：

1. 前端状态同步机制开发
2. 代理层框架、缓存、限流、熔断与降级能力开发

在获得 Coze 平台持久化存储边界确认前，**不建议**进行以下动作：

1. 接入未脱敏的真实生产业务数据
2. 面向真实业务用户开放试运行
3. 将 AI 助手作为正式生产能力发布
