## 1. 概述

### 1.1 背景

采购经营分析驾驶舱 Phase 1 已完成 8 大模块、37 个 ECharts 图表、10 个 KPI 和 6 条 P0 预警的可视化建设。现阶段缺口不在“有没有数据”，而在“用户是否能快速得到可执行结论”。

v1 设计已验证 AI 集成方向可行，但在以下方面存在高风险：

- 页面上下文仅在发送消息时采集，状态容易漂移
- 将整页 KPI 和预警全量注入 system prompt，存在 token、准确率和数据外发风险
- 代理层缺少明确的超时降级、高可用和缓存策略
- 业务口径文档不足以支撑追问型场景
- 日志治理和第三方数据边界定义不完整

v2 的目标是把以上问题在设计阶段先收口，再进入开发。

### 1.2 目标

通过 Coze Bot + 后端代理 + 最小上下文注入 + 按需取数机制，实现以下能力：

1. **智能问数**：支持围绕 8 大模块和 37 个图表的自然语言问答。
2. **智能解读**：基于当前视图和业务规则生成结论、原因和建议。
3. **上下文感知**：AI 感知当前模块、筛选器、选中对象和图表焦点。
4. **口径可追溯**：涉及降本、TCO、预警、供应商等问题时，可回溯到业务口径文档。
5. **安全可控**：敏感数据最小化传输，日志可脱敏审计，第三方数据边界明确。

### 1.3 范围

**本阶段覆盖**

- 驾驶舱页面内嵌 AI 助手组件
- 支持 8 大模块、37 个图表的智能问答与解读
- 支持 10 个 KPI、6 条 P0 预警的上下文问答
- 支持快捷提问、回答引用来源、页面联动提示
- 支持最小上下文注入、按需取数、缓存、超时降级

**本阶段不覆盖**

- 自动发起采购流程或审批流
- 跨系统写操作，例如自动补货、自动下单
- 开放式多代理协同分析
- 移动端独立 App

## 2. 设计原则

### 2.1 最小必要原则

不向模型全量发送整页数据，只发送回答当前问题所需的最小上下文。

### 2.2 状态优先原则

AI 的页面理解必须来源于实时同步状态，而不是发送消息瞬间的单次抓取。

### 2.3 口径先于回答

涉及金额、比例、价格偏差、TCO、预警判断等问题时，先遵循业务口径，再组织回答。

### 2.4 降级优先原则

当 Coze、网络或代理异常时，优先给用户稳定反馈，而不是长时间空等或返回不透明错误。

### 2.5 安全默认收紧

默认不记录原始问题全文，不向第三方传输非必要业务数据，不暴露内部错误细节。

### 3.2 核心设计决策

#### 决策 1：采用“最小 system prompt + 按需工具取数”

不再把 `current_kpis`、`p0_alerts` 整页 JSON 直接塞进 system prompt。system prompt 只保留角色、回答规范、边界约束。实时业务数据由代理层工具接口按需返回。

#### 决策 2：在前端增加状态同步总线

新增 `dashboardStateBus`，统一接收 Tab 切换、筛选器变化、图表点击和 KPI 高亮事件。AI 助手使用总线中的最新状态，而不是每次现抓 DOM。

#### 决策 3：代理层承担降级控制

AI 代理不仅转发请求，还负责缓存、限流、熔断、trace_id、脱敏日志和错误标准化。

#### 决策 4：业务口径作为独立知识层

把降本、物流、返利、TCO、价格偏差、预警责任等定义从图表文档中抽离成独立规则文档，避免回答时口径漂移。

## 4. 前端设计

### 4.1 组件形态

AI 助手继续采用右下角悬浮组件，但增加 3 种状态：

1. `collapsed`: 仅保留入口按钮
2. `dock`: 380 x 500 对话窗口
3. `focus`: 临时扩展高度，适合长回答阅读

组件默认采用深色科技风，颜色变量沿用驾驶舱设计系统，但命名统一为 `--pa-ai-*` 前缀，避免样式污染。

### 4.2 页面状态同步

#### 4.2.1 状态对象 schema

```javascript
{
  session: {
    userRole: 'procurement_manager',
    dateRange: '2026-W15'
  },
  ui: {
    activeTab: 'tab2',
    activeModule: '支出分析',
    selectedDimension: 'category',
    activeFilters: {
      bu: '事业部3',
      subsidiary: null,
      category: '钢材'
    },
    selectedEntity: {
      type: 'chart',
      id: 'chartPriceDeviation'
    },
    highlightedChart: 'chartPriceDeviation'
  }
}
```

#### 4.2.2 状态更新事件

以下事件必须触发状态同步：

- Tab 切换
- 维度切换
- KPI 卡片点击
- 图表点击或 hover 高亮
- 筛选器变化
- AI 触发的页面跳转或高亮操作

#### 4.2.3 前端接口设计

```javascript
dashboardStateBus.update(partialState)
dashboardStateBus.getSnapshot()
dashboardStateBus.subscribe(listener)
```

AI 组件发送消息前只读取 `getSnapshot()`，不直接遍历 DOM 现抓状态。

### 4.3 前端生命周期与性能

#### 4.3.1 生命周期要求

- 页面初始化时创建 AI 助手单例
- 页面卸载时销毁 SDK 实例和订阅监听器
- Tab 切换时不重建 AI 组件，只更新状态
- AI 高亮图表前先校验图表实例未销毁

#### 4.3.2 内存与性能约束

- 不得在每次提问时重新初始化 SDK
- 不得在每次回答后重复绑定事件
- 不得在不可见图表上频繁执行高亮动画
- AI 组件对页面帧率影响需控制在可接受范围内

#### 4.3.3 大屏适配

- 1920 固定宽度下默认停靠右下角
- 小屏或浏览器缩放低于阈值时自动切换为 `collapsed`
- 避免覆盖右侧关键图表区域；若冲突，优先收起对话框

### 4.4 快捷提问重构

快捷按钮从“演示型按钮”改为“晨会型按钮”，建议保留 4 类：

1. `本周最差 3 项指标`
2. `当前 P0 预警怎么处理`
3. `这张图的结论和原因`
4. `给我一条晨会汇报口径`

## 5. 对话与 Prompt 设计

### 5.1 Prompt 分层

#### 5.1.1 System prompt

只保留长期稳定规则：

- 角色：采购经营分析驾驶舱 AI 助手
- 输出语言：中文
- 格式规范：金额、比例、来源标注
- 边界约束：不确定时明确说明，禁止编造数据
- 回答顺序：结论、原因、建议、来源

#### 5.1.2 Runtime context

只包含当前会话必要状态：

- 当前模块
- 当前 Tab
- 当前筛选器摘要
- 当前选中对象摘要

#### 5.1.3 Tool data

当用户问题涉及具体 KPI、预警或图表明细时，由代理调用工具接口返回结构化数据。

### 5.2 推荐回答模板

```markdown
结论：本月降本额为 156.8 万元，当前达成率 108.1%。

原因：钢材谈判降幅扩大，同时事业部 3 集中采购带来价格优势。

建议：优先跟进塑料粒子缺货风险，并补充核对该口径是否包含物流费用。

来源：降本分析模块 / 月度降本追踪图 / 降本口径文档 v1
```

### 5.3 幻觉控制规则

当出现以下情况时，必须禁止模型直接给结论：

- 实时数据缺失
- 当前口径文档不存在
- 用户问题跨模块但未给足筛选条件
- 实时数据与知识库示例冲突

此时回答策略为：

1. 明确说明缺失项
2. 告知需要切换模块或补充条件
3. 只引用已确认的数据和来源

## 6. 工具与数据访问设计

### 6.1 工具接口定义

代理层提供以下工具能力给 Coze：

1. `get_dashboard_kpis`
2. `get_dashboard_alerts`
3. `get_chart_snapshot`
4. `get_business_glossary`
5. `get_faq_answer`

### 6.2 示例：图表快照工具

```json
{
  "tool": "get_chart_snapshot",
  "input": {
    "chart_id": "chartMonthlySavings",
    "active_tab": "tab1",
    "filters": {
      "bu": "事业部1"
    }
  }
}
```

### 6.3 工具返回约束

- 返回结构化 JSON，不返回冗长自然语言
- 金额、比例、单位字段必须拆开
- 必须带 `source_id` 和 `version`
- 缺失数据时显式返回 `not_available`

## 7. 后端代理设计

### 7.1 服务职责

代理层职责定义如下：

1. JWT 鉴权和来源控制
2. Prompt 组装与工具路由
3. 缓存、限流、熔断、超时降级
4. 脱敏日志、trace_id、监控埋点
5. 错误码标准化

### 7.2 高可用建议

- 多实例部署
- 负载均衡入口
- 健康检查接口
- 缓存使用 Redis
- 限流与熔断状态共享

### 7.3 超时与降级策略

| 场景 | 阈值 | 处理策略 | 前端提示 |
|------|------|----------|---------|
| Coze 响应慢 | > 5 秒 | 展示思考中状态 | “正在分析，请稍候” |
| Coze 超时 | > 10 秒 | 进入降级模式 | “AI 分析超时，先返回简版结论” |
| 连续失败 | 5 次 / 5 分钟 | 熔断 30 秒 | “AI 服务暂不可用，请稍后重试” |
| 高频标准问题 | 命中缓存 | 返回缓存答案 | “已返回最近一次分析结果” |

### 7.4 缓存策略

适合缓存的问题：

- KPI 定义类问题
- 图表解释类问题
- FAQ 标准问题
- 更新频率较低的晨会总结问题

不适合缓存的问题：

- 当前筛选器下的实时追问
- 刚触发的预警处理状态
- 带用户个性化上下文的问题

缓存键建议：

`sha256(question + active_tab + filters + glossary_version)`

### 7.5 可观测性

每次请求必须打通统一的 `trace_id`，并记录：

- 请求时间
- 响应时间
- 命中模块
- 工具调用次数
- 缓存是否命中
- 是否触发降级
- 用户反馈结果

## 8. 安全与合规设计

### 8.1 数据最小化

- 禁止发送完整页面状态到第三方平台
- 禁止默认发送全部 KPI 与全部预警详情
- 仅在需要时发送与问题直接相关的数据片段

### 8.2 密钥与身份认证

- Coze API Key 存于服务端密钥管理系统或环境变量
- 建议后续接入企业级 KMS / Vault
- JWT 必须包含过期时间，并限制前端存储策略
- 启用 CORS 白名单、HTTPS、HSTS

### 8.3 日志治理

默认日志不得记录：

- 原始问题全文
- 完整 KPI 快照
- 完整预警详情
- 供应商敏感标识

日志需记录：

- trace_id
- 用户 ID
- 模块
- 问题分类
- 耗时
- 错误类型

日志需增加以下控制：

- 脱敏
- 分级访问
- 留存周期
- 审计记录

### 8.4 第三方数据边界

在上线前必须确认：

1. Coze 平台是否使用输入数据参与模型训练
2. 数据是否跨境存储或处理
3. 业务敏感数据是否需要额外授权
4. 用户是否需要看到隐私提示

## 10. 业务口径要求

以下问题必须在知识库中有明确答案：

1. 降本额是否包含物流费用
2. 降本额是否包含返利、税费、包装费用
3. 上期单价如何定义
4. TCO 包含哪些费用项
5. 价格偏离的市场价来源和更新频率
6. P0 预警是否包含责任人和处理状态
7. 到货承诺来自哪个系统和哪个字段

如果以上任一问题无明确口径，AI 必须显式回答“当前口径未定义”。

## 11. 示例交互

### 11.1 事实查询

**用户**：本月降本额是多少？  
**AI**：结论：本月降本额为 156.8 万元。来源：降本分析模块 / 月度降本追踪图。当前口径按“采购价差 x 本期采购量”计算，不含物流费用。

### 11.2 图表追问

**用户**：这张图为什么没达标？  
**AI**：结论：当前图表显示塑料粒子相关降本进度落后。原因：预算降本额未完成，且市场价格回弹。建议：切换到价格趋势模块，核对塑料粒子的市场公允价与锁价策略。来源：当前高亮图表 + 价格偏离口径文档。

### 11.3 口径追问

**用户**：这包含物流费用吗？  
**AI**：当前回答中的降本额不包含物流费用。依据：业务口径文档 `cost-calculation.md`。如果你需要全成本口径，请切换到支出分析模块查看 TCO 相关指标。
